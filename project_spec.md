# Heimdall 프로젝트 명세서

## 1. 프로젝트 개요

**Heimdall**은 AI를 활용해 사용자 간 토론을 중재하고 판정하는 실시간 토론 앱 서비스이다.

사용자는 토론 주제를 선택하거나 직접 생성한 뒤, 정해진 토론 구조에 따라 상대방과 발언을 주고받는다. 토론이 종료되면 AI가 전체 토론 내용을 분석하여 각 발언의 주장, 근거, 반박, 질문, 답변 관계를 구조화하고, 사실 검증과 논리 평가를 바탕으로 최종 승자와 피드백을 제공한다.

이 프로젝트의 핵심은 단순한 채팅 서비스가 아니라, **토론 과정을 기록하고 논증 구조를 분석한 뒤 공정한 기준으로 판정하는 AI 중재 시스템**을 구현하는 것이다.

---

## 2. 프로젝트 배경

기존 온라인 토론은 사용자의 감정적 표현, 논점 이탈, 근거 없는 주장, 말싸움으로 흐르기 쉽다. 또한 토론이 끝난 뒤 누가 더 논리적으로 주장했는지, 어떤 근거가 설득력 있었는지 객관적으로 판단하기 어렵다.

Heimdall은 이러한 문제를 해결하기 위해 AI를 토론의 심판 역할로 활용한다.

AI는 사용자의 발언을 단순히 요약하는 것이 아니라, 다음과 같은 요소를 중심으로 토론을 분석한다.

- 발언에 명확한 주장이 있는지
- 주장을 뒷받침하는 근거가 존재하는지
- 상대방의 주장에 적절히 반박했는지
- 질문과 답변이 논점에 맞게 이어졌는지
- 사실 확인이 필요한 주장이 있는지
- 전체 토론 흐름에서 어느 쪽의 논증이 더 탄탄한지

이를 통해 사용자는 토론 결과뿐 아니라 자신의 논리적 강점과 약점도 확인할 수 있다.

---

## 3. 서비스 목표

Heimdall의 목표는 다음과 같다.

1. **구조화된 토론 경험 제공**  
   자유로운 말싸움이 아니라 입론, 반론 및 질문, 최종 발언으로 이어지는 명확한 토론 흐름을 제공한다.

2. **AI 기반 공정 판정 제공**  
   토론 종료 후 AI가 전체 발언을 분석하고, 논리성·근거성·반박력·일관성 등을 기준으로 승자를 판정한다.

3. **논증 구조 시각화 기반 마련**  
   발언을 주장, 근거, 반박, 질문, 답변 단위로 분리하고 관계를 그래프 형태로 저장하여 이후 시각화와 피드백에 활용한다.

4. **사실 검증을 포함한 토론 평가**  
   사실 확인이 필요한 주장에 대해 별도 Fact Check를 수행하고, 최종 판정에 반영한다.

5. **토론 학습 도구로 확장 가능성 확보**  
   사용자가 토론 후 자신의 발언 습관, 논리 전개 방식, 근거 제시 능력을 개선할 수 있도록 피드백을 제공한다.

## 4. 핵심 사용자 시나리오

### 4.1 토론방 생성

사용자는 토론 주제를 선택하거나 직접 입력하여 토론방을 생성한다.

예시 주제:

- 대학의 출석 점수는 성적에 반영되어야 하는가?
- AI 면접은 공정한 채용 방식인가?
- 원격 근무는 생산성을 높이는가?
- 동물 실험은 금지되어야 하는가?

토론방 생성 시 사용자는 다음 정보를 설정할 수 있다.

- 토론 주제
- 찬성/반대 입장
- 라운드 수
- 공개 여부
- 상대방 초대 또는 매칭 방식

---

### 4.2 토론 참여

사용자는 생성된 토론방에 입장해 상대방과 토론을 진행한다.

토론은 정해진 순서에 따라 진행되며, 각 사용자는 자신의 턴에만 발언할 수 있다.

기본 토론 흐름은 다음과 같다.

1. 입론
2. 반론 및 질문
3. 반론 및 질문 반복
4. 최종 발언
5. AI 판정

---

### 4.3 토론 진행

토론은 발언 단위로 진행된다.

각 발언에는 제한 시간이 존재하며, 사용자는 제한 시간 안에 자신의 주장을 작성해야 한다.

기본 시간 제한:

- 입론: 1분
- 반론 및 질문: 3분
- 최종 발언: 1분

각 발언은 최대 1,000자까지 입력할 수 있다.

---

### 4.4 AI 분석

토론이 종료되면 AI 분석이 시작된다.

AI는 토론 도중 실시간으로 판정하지 않고, 토론이 끝난 뒤 전체 흐름을 바탕으로 분석한다.

AI 분석은 다음 단계로 구성된다.

1. **Analyzer**  
   사용자의 발언을 주장, 근거, 반박, 질문, 답변 등으로 분리하고 논증 구조를 생성한다.

2. **Fact Checker**  
   Analyzer가 사실 확인이 필요하다고 판단한 주장만 대상으로 비동기 사실 검증을 수행한다.

3. **Judge**  
   논증 구조와 Fact Check 결과를 바탕으로 최종 점수, 승자, 피드백을 생성한다.

---

### 4.5 판정 결과 확인

사용자는 토론 종료 후 AI 판정 결과를 확인할 수 있다.

판정 결과에는 다음 내용이 포함된다.

- 최종 승자
- 양측 점수
- 판정 근거
- 주요 주장 요약
- 강점 분석
- 약점 분석
- 사실 검증 결과
- 개선 피드백

## 5.4 AI 분석 기능

### Analyzer

Analyzer는 토론 종료 후 전체 발언을 분석한다.

주요 역할:

- 발언에서 주장 추출
- 근거 추출
- 반박 관계 추출
- 질문과 답변 관계 추출
- 논증 구조 생성
- 사실 검증이 필요한 주장 표시

Analyzer의 결과는 Argument Graph 형태로 저장된다.

---

### Argument Graph 생성

토론 내용은 단순 텍스트가 아니라 논증 구조로 변환된다.

그래프 구성 요소:

- 주장
- 근거
- 반박
- 질문
- 답변
- 보충 설명
- 사실 검증 대상 주장

그래프 관계 예시:

- A 주장이 B 근거에 의해 뒷받침됨
- A 주장이 B 주장에 의해 반박됨
- A 질문이 B 답변으로 이어짐
- A 주장이 Fact Check 대상으로 표시됨

---

### Fact Checker

Fact Checker는 Analyzer 결과 중 `requiresFactCheck = true`인 항목만 대상으로 실행된다.

주요 역할:

- 사실 확인이 필요한 주장 선별
- 외부 검색 또는 신뢰 가능한 정보 기반 검증
- 사실 여부 판단
- 판정에 참고할 검증 결과 생성

Fact Check는 전체 발언을 대상으로 무조건 수행하지 않고, 필요한 주장에 대해서만 비동기적으로 수행한다.

---

### Judge

Judge는 최종 판정을 담당한다.

입력 정보:

- 전체 토론 구조
- Argument Graph
- Fact Check 결과
- 토론 메타데이터
- 양측 발언 기록

판정 기준:

- 주장 명확성
- 근거의 타당성
- 반박의 적절성
- 논리적 일관성
- 질문과 답변의 적합성
- 사실 오류 여부
- 전체 설득력

출력 정보:

- 최종 승자
- 양측 점수
- 판정 사유
- 주요 근거
- 사용자별 강점
- 사용자별 약점
- 개선 피드백

## 8. 프로젝트 핵심 차별점

Heimdall의 차별점은 다음과 같다.

1. **단순 채팅이 아닌 구조화된 토론 서비스**  
   자유 채팅이 아니라 입론, 반론 및 질문, 최종 발언이라는 토론 흐름을 제공한다.

2. **AI가 토론 종료 후 전체 맥락을 보고 판정**  
   중간중간 즉시 평가하지 않고, 전체 토론이 끝난 뒤 문맥을 반영하여 판정한다.

3. **Argument Graph 기반 분석**  
   발언을 단순 요약하지 않고 주장, 근거, 반박, 질문, 답변 관계로 구조화한다.

4. **필요한 주장만 Fact Check 수행**  
   모든 문장을 검증하지 않고, 사실 확인이 필요한 주장만 선별해 검증한다.

5. **승패뿐 아니라 학습 피드백 제공**  
   단순히 누가 이겼는지 알려주는 것이 아니라, 왜 이겼는지와 어떻게 개선할 수 있는지를 제공한다.
